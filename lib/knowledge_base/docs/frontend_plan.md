# 小说生成前端规划

## 目标

把当前以 `run_novel_generation.py` 为中心的命令行流程，扩展为一个本地可视化工作台，让用户能直接创建项目、调整写作参数、发起生成、查看章节和衍生内容，而不是记忆命令参数。

## 设计原则

1. **先做本地工作台，再考虑远程服务**
2. **前端只编排，不重复实现生成逻辑**
3. **所有风格/写法参数复用 `writing_options.py`**
4. **项目状态、生成结果、检查点继续复用现有 JSON 文件**

## 推荐技术路线

### 第一阶段：Streamlit 本地工作台

原因：
- 仓库当前以 Python 为主，没有稳定前端工程
- 可以最快把 CLI 包装成可视界面
- 适合本地运行、表单参数输入、日志展示、文件预览

建议页面：
- `项目页`：创建/加载项目，展示当前进度、基础设定
- `写作参数页`：风格预设、视角、节奏、对白密度、战斗写法等
- `生成页`：起始章节、数量、dry-run、自动反馈开关、实时日志
- `内容页`：章节列表、章节正文、plot summary、一致性报告
- `衍生页`：播客、视频提示词、角色描述

### 第二阶段：FastAPI + React/Vite

适用时机：
- 需要更细的交互状态
- 需要多标签工作流
- 需要更好的阅读器、diff、时间线、批量操作

建议拆分：
- FastAPI：对现有 CLI/agent 层做薄 API 封装
- React/Vite：构建小说工作台

## 页面信息架构

### 1. 项目概览
- 当前项目
- 总章节 / 已生成章节
- 最近一次生成结果
- 当前写作参数摘要

### 2. 写作控制台
- 基础风格
- 风格预设
- 叙事视角
- 叙事写法
- 节奏
- 对白密度
- 行文质感
- 设定密度
- 情绪强度
- 战斗写法
- 开篇抓力

附加能力：
- 保存为“项目默认参数”
- 保存为“预设方案”
- 一键恢复默认

### 3. 章节生成面板
- 起始章节
- 生成数量
- 断点续传
- Dry-run
- 自动反馈开关
- 实时日志流
- 成功/失败统计

### 4. 章节阅读面板
- 左侧章节目录
- 中间正文阅读
- 右侧结构信息：
  - 章节摘要
  - 关键事件
  - 角色状态
  - 一致性报告

### 5. 衍生内容面板
- 视频 Prompt
- 播客脚本
- 角色描述
- 场景描述

## 后端接口草案

即便第一阶段先做 Streamlit，也建议把数据访问收敛成函数级接口，后续方便升到 API：

- `list_projects()`
- `create_project(payload)`
- `load_project(project_id)`
- `get_project_status(project_id)`
- `get_writing_options(project_id)`
- `update_writing_options(project_id, payload)`
- `generate_chapters(project_id, payload)`
- `list_chapters(project_id)`
- `get_chapter(project_id, chapter_number)`
- `list_derivatives(project_id)`

## 实施顺序

1. 已完成：统一写作参数定义，抽到 `writing_options.py`
2. 已完成：CLI 和主生成链路支持写作参数
3. 下一步：补一个 `streamlit_app.py`
4. 然后：把生成日志和章节列表接到 UI
5. 最后：增加章节阅读和衍生内容面板

## 最小可交付版本

一个本地网页，能完成以下事情：
- 创建/加载项目
- 调写作参数
- 点击按钮生成章节
- 查看生成日志
- 查看章节列表和正文

这一步做完，命令行就不再是主入口，只保留给自动化和调试使用。
